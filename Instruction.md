 ## 2. Установка git
 Загружаем  последнюю версию git с сайта https://git-scm.com/downloads.
 Устанавливаем с настройками умолчанию.
 ## 3. Настройка git

 При первом использовании git необходимо представиться. Для этого нужно ввести в терминале две команды:
 ```sql
 git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```
## 4. Инициализация репозитория

Для инициализации необходимо ввести команду **git init**

## 5. Начало работы

Теперь указать расположение нашего репозитория в левом углу экрана, и там же слева создать новый файл с указанием его расширения. И начать его отслеживание git вводом команду git add "Имя файла"

## 6. Первые операции

Теперь Git отслеживает изменения файлов вашего проекта. Но, так как вы только создали репозиторий в нем нет вашего кода. Для этого необходимо создать commit, и можно записывать изменения нашего проекта пошагово, используя возможности ***git***. После каждого изменения проекта есть возможность сохранить именно это его состояние вводом команды  *git add "Имя файла"* 
И git commit -m **"Первый комментарий"** В каждом новом commit следует оставлять коммент и в нем описывать суть изменений.

## 7. Просмотр истории коммитов

Если есть необходимость вернуться на раннюю версию , то вводим 

```C#
git log

```
Эта команда выводит все ветки изменений и их имена, например

commit ***4d93***acfa29a64f181dab3a14f70aeb2c0579c42a (HEAD -> master)
Author: samat <samatsr@gmail.com
Чтобы вернуться к ней необходимо ввести команду git checkout "Первые четыре символа" нужного нам коммита.

## 8. Несколько самых важных команд нееобходимых для работы 

Если вы забыли подписать последние изменения и не помните на чем закончили работу, то можно воспользоваться командой 
```
git diff
```
Эта команда выводит информацию о самых последних изменениях.

Красным цветом удаленные, зеленым цветом добавленные строки.
И команда git status , с помощью которой можно узнать есть ли несохраненные данные в нашем проекте.

## 9 Инициализация репозитория

Перед началом работы необходимо инициализировать git.
Первым делом необходимо выполнить команду 
*git init*.
Для этого необходимо указать программе на нужную папку и после этого начинать работать в ней.

## 10 Запись изменений в репозиторий
 
 После того как были внесены изменения в репозиторий, необходимо их сохранить. Это можно сделать несколькими способами, например:
 ```
  git add "Имя нашего файла", git commit -m "Комментарий проделанных изменений" 
  ```
  или
  ```
  git commit -am "Комментарий проделанных изменений"
  ```
  эта команда позволяет и закоммитить и оставть комментарий одновременно

  ## 13 Игнорирование файлов

  Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки необходимо создать там файл ***.gitignore*** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.

  ## 14 Создание веток в git

   Ветка в git - это простой перемещаемый указатель на один из коммитов, обычно последний в цепочке коммитов.
   По умолчанию имя основной ветки в git - **master**. И после этого вносить изменения в эту ветку.

Список веток в репозитории можно посмотреть с помощью команды `git branch`.
Текущая ветка будет отмечена звёздочкой:
**\*master**
Создать можно командой:
```
git branch "имя новой ветки"
```
в результате создаётся новый указатель на текущий коммит.

## 15. Слияние веток и разрешение конфликтов 

Для слияния выбранной ветки с текущей, нужно выполнить команду:
```
git merge "название выбранной ветки"
```

 ## 16. Удаление веток в git

Для того чтобы удалить ветку необходимо выполнить команду 
```
git branch -d "Имя удаляемой ветки"
```
при этом находиться на удаляемой ветке нельзя, иначе git не даст это сделать.
 Если есть необходимость удалить ветку без слияния (принудительно) , то нужно выполнить команду 
 ```
 git branch -D "Имя удаляемой ветки", 
``` 
 и так же нужно находиться вне неё. 

В случае когда в сливавемых ветках есть расхождения в их содержании, git выводит сообщение о конфликте которое нужно разрешить вручную.
Git приедлгает решить это несколькими способами:

* оставить вариант из ветки master
* принять изменения из слияемой ветки
* оставить информацию из master и входящую 
* сравнить изменения

`ВАЖНО` чтобы такие конфликты разрешал всегда только один человек.
